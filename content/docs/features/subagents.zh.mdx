---
title: 子智能体与编排
description: 智能体可派生专职助手,并用代码模式批量调用工具。
---

对于更大的任务,智能体可将工作拆分给**子智能体(subagent)**,并用**编排
(orchestration)**高效地批量调用工具。

## 子智能体(角色)

智能体可派生专注的助手,每个助手都有一个**角色(role)**,定义其模式、可用工具和
迭代预算。内置角色包括:

| 角色 | 模式 | 适合 |
| --- | --- | --- |
| **explorer** | browser | 跨页面快速侦察 |
| **researcher** | browser | 更深入的多步网络研究 |
| **worker** | agent | 完成具体任务 |
| **reader** | agent | 阅读并从内容中提取 |

主智能体派生它们,等待结果(单个或全部),检查状态,并可停止它们。角色是带 frontmatter
的 Markdown 文件,因此你可以添加自己的角色。

*示例:* "研究排名前 3 的向量数据库并总结取舍" —— 智能体可能为每个数据库派生一个
**researcher**,再汇总它们的发现。

## 编排(代码模式)

智能体不必逐个发出工具调用,而是可以用 **`orchestrate`** 编写一段简短的 Python 脚本,
一次性调用多个工具。脚本在**沙箱解释器**中运行(禁止 import、文件系统,以及所提供
工具之外的网络),并带有自动检查与修复。对于重复性的多步工作(如访问 20 个页面并从
每个页面采集一个字段),这要快得多。

如果你想在运行前审阅,智能体会先生成一份**计划(plan)**。
